# frame-jank — reproduction artifacts

Everything needed to regenerate the screenshots in
[`docs/case-studies/frame-jank.md`](https://github.com/fiveapplesonthetable/perfetto/blob/heap-docs-screenshots/docs/case-studies/frame-jank.md)
from scratch.

## Layout

```
frame-jank/
├── README.md                  (this file)
├── demo-buggy/                ListView + sync BitmapFactory.decodeByteArray on UI thread
│   ├── AndroidManifest.xml
│   ├── assets/                5 × 2048×2048 PNGs (regenerated by python helper)
│   ├── build.sh               javac → d8 → aapt2 → zipalign → apksigner
│   └── src/com/example/perfetto/jank/JankActivity.java
├── demo-fixed/                same UI; LruCache + Executors.newSingleThreadExecutor
│   └── src/com/example/perfetto/jank/JankActivity.java
├── trace-configs/jank.cfg     atrace gfx/view/sched + frametimeline + ftrace sched
├── traces/
│   ├── before.pftrace         buggy app, 12 s window
│   └── after.pftrace          fixed app, 12 s window
├── screenshots/               raw Playwright captures (superset of what landed)
└── playwright/shoot.js        captures the doc's three screenshots
```

## One-shot reproduction

Assumes a running cuttlefish at `adb devices` and the AOSP checkout
at `~/dev/aosp` (the build script uses prebuilt aapt2/d8/apksigner +
android.jar API 36 + the platform testkey).

```bash
# 1. Build, install, capture buggy trace.
cd frame-jank/demo-buggy
./build.sh
adb install -r build/jankdemo.apk
adb shell am force-stop com.example.perfetto.jank
adb shell am start -n com.example.perfetto.jank/.JankActivity
sleep 2                                  # auto-scroll starts at +1.5 s

adb push ../trace-configs/jank.cfg /data/local/tmp/
adb shell perfetto --txt -c /data/local/tmp/jank.cfg \
    -o /data/local/tmp/before.pftrace
adb pull /data/local/tmp/before.pftrace ../traces/before.pftrace

# 2. Build, install, capture fixed trace.
cd ../demo-fixed
./build.sh
adb install -r build/jankdemo.apk
adb shell am force-stop com.example.perfetto.jank
adb shell am start -n com.example.perfetto.jank/.JankActivity
sleep 2

adb shell perfetto --txt -c /data/local/tmp/jank.cfg \
    -o /data/local/tmp/after.pftrace
adb pull /data/local/tmp/after.pftrace ../traces/after.pftrace

# 3. Verify the contrast on both traces.
TPS=~/dev/aosp/out/host/linux-x86/bin/trace_processor_shell
for t in before after; do
  echo "=== $t ==="
  echo "SELECT COUNT(*) AS over_16ms FROM slice WHERE name LIKE '%Adapter.getView' AND dur > 16000000;" \
    | $TPS -q /dev/stdin ../traces/${t}.pftrace 2>/dev/null
  echo "SELECT jank_type, COUNT(*) AS n FROM actual_frame_timeline_slice WHERE upid = (SELECT upid FROM process WHERE name = 'com.example.perfetto.jank') GROUP BY jank_type ORDER BY n DESC LIMIT 5;" \
    | $TPS -q /dev/stdin ../traces/${t}.pftrace 2>/dev/null
done

# 4. Re-shoot the doc's screenshots.
cd ../playwright
node shoot.js                            # writes to ../screenshots/
```

Step 3 should print:

| trace | getView slices > 16 ms | dominant jank_type |
|---|---|---|
| before | ~270 | "Prediction Error, App Deadline Missed, Buffer Stuffing" (~220) |
| after  | 0    | "Prediction Error, Buffer Stuffing" (cuttlefish noise; no app-side miss) |

| Source line | Screenshot | What it shows |

## Notes

- FrameTimeline data source needs **Android 12 (API 31) or higher**.
  Cuttlefish API 36 here is well above the minimum.
- The 2048×2048 PNG choice is deliberate: a smaller asset
  (256×256 or 512×512) decodes too fast to reliably cause jank on
  cuttlefish's variable-timing virtual display. 2048² lands every
  bind at 30+ ms and makes the jank deterministic.
- `Trace.beginSection("BadAdapter.getView")` in the demo is what
  surfaces the slice in the trace. Without it you'd still see jank
  in the Frame Timeline but you wouldn't have a labelled slice on
  the main thread to point at.
- Cuttlefish's frame timeline reports a constant background of
  `Prediction Error, Buffer Stuffing` jank that's an artefact of
  the virtual display, not the app. The signal that matters is the
  delta in frames flagged `App Deadline Missed` between before
  (~222) and after (~1).
- The Playwright shooter relies on a locally-built Perfetto UI on
  port 10000 (matches the heap-docs artifacts). Start it with
  `cd ~/dev/heapdocs/perfetto && ui/run-dev-server -n` before
  running `node shoot.js`.
